💰 Project SettleFlow - Final PRD (MBP Edition)
1. 프로젝트 개요 (Project Overview)
이름: SettleFlow (세틀플로우)

한줄 설명: 1인 커머스 사업자를 위한 구글 시트 기반의 교환/반품 및 거래처 정산 통합 관리 툴.

유형: MacBook Pro 로컬 실행형 Streamlit 웹 서비스.

난이도: 보통 (Google Sheets API 연동 및 상태 관리 로직).

2025 유효성: Google Sheets API v4 및 Python 3.11+ 환경에서 안정적 구동 확인.

2. 사용자 시나리오 (User Scenario)
대상: MacBook Pro로 모든 업무를 처리하는 1인 사업자(Doo).

시점:

CS 발생 시: 즉시 맥북에서 SettleFlow 실행 후 정보 입력.

거래처 소통 시: 생성된 정보를 카톡으로 전달.

정산 시: 거래처별 정산 단계 업데이트 및 누락 확인.

목적: 엑셀의 복잡함에서 탈출하고, 실시간으로 돈의 흐름(매입 채무)을 장악하기 위해.

3. 핵심 기능 목록 (Core Features)
[필수 기능 - Must Have]
교환/반품 통합 입력: 날짜, 상품명, 고객명, 연락처, 사유, 거래처, 송장번호, 택배비여부, 비고 등 9개 필드 수동 입력.

로지스틱스 상태 관리: [신청] → [수거신청] → [입고완료] → [환불완료] 버튼형 상태 변경.

카톡 전송용 복사: 거래처 사장님에게 보낼 핵심 정보를 한 줄 양식으로 클립보드에 즉시 복사.

거래처 정산 트래킹: [확인전] → [확인완료] → [세금계산서 발행 완료] → [입금완료] 4단계 프로세스 관리.

Google Sheets 연동: 모든 입력 데이터는 실시간으로 구글 스프레드시트에 저장 및 동기화.

데이터 필터링: 거래처별/상태별 검색 및 모아보기 기능.

[선택 기능 - Nice to Have]
대시보드 통계: 이번 달 총 반품 건수 및 미결제 정산 금액 합계 시각화.

엑셀 내보내기: 특정 기간 데이터를 별도 파일로 추출.

4. 기술 스택 (Technical Stack)
Language: Python 3.11+

Frontend/Web: Streamlit (로컬 호스트 실행)

Database: Google Sheets API v4 (gspread 라이브러리 활용)

Library: Pandas (데이터 정렬), Pyperclip (클립보드 복사)

5. 화면 구성 (UI Layout)
Tab 1: 교환/반품 센터

상단: 9개 필드 입력 폼 & [등록] 버튼.

하단: 현재 진행 중인 반품 리스트 (상태별 색상 구분).

액션: 행별로 [카톡복사], [상태업데이트] 버튼 배치.

Tab 2: 정산 매니저

상단: 거래처 등록 및 정산 금액 입력.

하단: 정산 프로세스 단계별 리스트 및 필터.

6. 상세 기능 명세 (Detailed Specs)
카톡 양식: [거래처명] 반품요청: {고객명}/{상품명}/{사유}/송장:{송장번호}

상태 변경: 버튼 클릭 시 구글 시트의 해당 행(Row) 데이터를 즉시 업데이트.

보안: API 인증용 credentials.json 파일은 프로젝트 폴더 내 안전하게 보관.

7. 디자인 가이드 (Design Guide)
색상 체계:

Blue (진행중), Orange (확인필요), Red (완료), Gray (대기).

가독성: 데이터가 많아질 경우를 대비해 한 페이지에 10~20건씩 끊어서 보여주는 페이징 혹은 스크롤 적용.

8. 제약 사항 (Constraints)
네트워크 의존성: 구글 시트를 사용하므로 인터넷 연결이 필수임.

수동 입력: API 자동화가 아니므로 Doo의 성실한 입력이 프로그램의 성능을 결정함.

API 한도: 구글 시트 API의 분당 읽기/쓰기 한도를 초과하지 않도록 로직 설계 (1인용으론 충분).
